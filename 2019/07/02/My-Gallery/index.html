2019-07-02 评论

SQL 分类

DDL: 数据库的定义，与数据库/表结构: create drop alter

DML: 数据操纵语言，操作表数据 insert update delete

DCL: 数据控制语言，设置用户的访问权限、安全 grant

DQL: 数据查询语言 select from where

列的约束

primary key 主键约束

默认就是不能为空, 唯一； 外键都是指向另外一张表的主键；主键一张表只能有一个

unique 唯一约束

列中的内容, 必须是唯一, 不能出现重复情况； 可以为空；可以有多个唯一约束；唯一约束不可以作为其它表的外键

not null 非空约束

where 条件后面

  • 关系运算符: >`` >= < <= !=[非标准sql语法] <>[标准sql语法]

  • 判断某一列是否为空: is null is not null

  • 在某范围内: in between...and

  • 逻辑运算符: and or not

  • 模糊查询: like
    _ : 代表单个字符
    % : 代表的是多个字符

  • 分组: group by

  • 分组之后条件过滤: having

  • 排序: order by (asc 升序, desc 降序)

常用聚合函数

sum() ,avg() , count() ,max(), min()

注意： where 条件后面不能跟聚合函数

编写顺序

S.. F.. W… G… H… O…

select … from … where … group by … having… order by …

执行顺序

F… W… G… H… S… O…

from … where … group by … select … order by …

delete 删除数据和 truncate 删除数据有什么区别

delete: DML 一条一条删除表中数据

truncate: DDL 先删除表再重建表

关于哪条执行效率高，具体要看表中的数据量。如果数据少， delete 比较高效；如果数据多，truncate 比较高效

where 与 having

having 可以接聚合函数，出现在分组之后

where 不可以接聚合函数，出现在分组之前
